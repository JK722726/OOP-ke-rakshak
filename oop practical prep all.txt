# EXPERIMENT A1 — ENGINEERING CALCULATOR

Concepts Used:

* Loops
* Conditional Statements
* Stack
* Exception Handling
* Methods
* Scanner Class

Main Objective:
Create a calculator that evaluates mathematical expressions dynamically.

Important Concepts:

## Scanner Class

Used to take input from user.

```java
Scanner sc = new Scanner(System.in);
```

## Infinite Loop

```java
while(true)
```

Program continuously accepts expressions until user types exit.

## Exception Handling

```java
try
catch
```

Used to prevent program crash.

## Stacks

```java
Deque<Double> values
Deque<Character> operators
```

Used for storing numbers and operators.

## Operator Precedence

```java
+
-
*
/
^
```

Multiplication/division have higher priority than addition/subtraction.

Important Methods:

### evaluateExpression()

Processes the entire expression.

### applyOperation()

Performs calculation.

### precedence()

Checks operator priority.

### isOperator()

Checks whether character is operator.

Flow of Program:

1. User enters expression
2. Numbers pushed into values stack
3. Operators pushed into operators stack
4. Operations performed according to precedence
5. Final result displayed

Important Viva Questions:

Q. Why use stack?
A. Stack helps evaluate expressions according to operator precedence.

Q. Why exception handling?
A. To handle invalid expressions and division by zero.

Q. Which exception handles divide by zero?
A. ArithmeticException.

Q. Which loop is used?
A. while loop.

Q. Why Scanner used?
A. For dynamic user input.

Important Keywords:

* Stack
* Deque
* Exception Handling
* Operator Precedence
* Loop

What to Say to External:

“This program implements a dynamic engineering calculator using stacks and exception handling. It supports arithmetic operations with operator precedence and parentheses handling.”

Classes Used:

Main Objective:
Create an e-commerce billing system with invoice generation.

Same constructor name but different parameters.

```java
Product(String name, double price)
Product(String name, double price, int quantity)
Product(String name, double price, int quantity, String category)
```

This provides flexibility while creating objects.

## Encapsulation

```java
private String name;
```

Data hiding using private variables.

## ArrayList

```java
List<Product> productList = new ArrayList<>();
```

Used for dynamic storage of products.

## Methods Used

### addProduct()

Adds product into order.

### calculateTotal()

Calculates total price.

### applyDiscount()

Applies discount based on conditions.

### generateInvoice()

Prints invoice.

Discount Logic:

* Above 5000 → 15%
* Above 3000 → 10%
* Above 1000 → 5%

Flow of Program:

1. Create products
2. Add products into ArrayList
3. Calculate total
4. Apply discount
5. Generate invoice

Important Viva Questions:

Q. What is constructor overloading?
A. Multiple constructors with same name but different parameters.

Q. Why ArrayList used?
A. Dynamic storage.

Q. Difference between array and ArrayList?
A. Array size fixed, ArrayList dynamic.

Q. Why private variables used?
A. Encapsulation and data security.

Important Keywords:

* Constructor Overloading
* ArrayList
* Encapsulation
* Invoice Generation

Same method name with different parameters.


Compiler decides which method to call.


Can be called without object.

## Math Class

```java
Math.pow()
Math.abs()
```

Used for mathematical calculations.

Flow of Program:

1. User enters values

Q. What is method overloading?

Q. Which polymorphism used?

Q. Can methods overload by return type only?
A. No.

Q. Why static methods?
A. Can be accessed without creating object.

Important Keywords:

* Compile-time Polymorphism
* Overloading
* Static Method
* Math Class

What to Say to External:

“This program demonstrates compile-time polymorphism through method overloading. Different power() and absolute() methods work with different data types.”

* Shape

## Interface

```java
interface Shape
```

Contains abstract methods.

## Methods

```java
void draw();
double calculateArea();
```

## Implementing Interface

```java
class Circle implements Shape
```

```java
Shape s1 = new Circle(5);
```

```java
s.draw();
```

Correct method selected at runtime.

A = πr²

Rectangle Area Formula:

Q. What is runtime polymorphism?
A. One reference, many forms.

Q. Can interface have object?
A. No.

Important Keywords:

* Interface
* implements
* Runtime Polymorphism
* Abstraction
* Dynamic Binding

# EXPERIMENT B3 — ABSTRACT CLASS

Concepts Used:

* Abstract Class
* Abstract Method
* Inheritance
* Runtime Polymorphism
* Method Overriding

Classes Used:

* Employee
* Developer
* Manager

Main Objective:
Demonstrate abstract classes and abstract methods.

## Abstract Class

```java
abstract class Employee
```

Cannot create object directly.

## Abstract Method

```java
public abstract double calculateBonus();
```

No body.
Subclass must implement.

## Concrete Method

```java
public void displayDetails()
```

Normal method with body.

## Child Classes

```java
class Developer extends Employee
class Manager extends Employee
```

## Runtime Polymorphism

```java
Employee emp1 = new Developer()
```

Flow of Program:

1. Employee reference created
2. Developer/Manager objects assigned
3. calculateBonus() runs according to object type

Important Viva Questions:

Q. What is abstract class?
A. Partial blueprint containing abstract and concrete methods.

Q. Can abstract class have constructor?
A. Yes.

Q. Can we create object of abstract class?
A. No.

Q. Difference between interface and abstract class?
A. Abstract class supports concrete methods and constructors.

Q. Why abstract method used?
A. To force subclasses to provide implementation.

Important Keywords:

* abstract
* Abstract Method
* Partial Abstraction
* Runtime Polymorphism

What to Say to External:

“This program demonstrates abstract class functionality. Employee is an abstract class containing both abstract and concrete methods. Developer and Manager provide their own implementations for calculateBonus().”

# VERY IMPORTANT COMMON VIVA QUESTIONS

## What is OOP?

Object Oriented Programming is programming based on objects and classes.

## Four Pillars of OOP

1. Inheritance
2. Polymorphism
3. Encapsulation
4. Abstraction

## Difference Between Overloading and Overriding

| Overloading                           | Overriding                  |
| ------------------------------------- | --------------------------- |
| Same method name different parameters | Same method same parameters |
| Compile time                          | Runtime                     |
| Same class                            | Parent-child class          |

## Difference Between Interface and Abstract Class

| Interface         | Abstract Class      |
| ----------------- | ------------------- |
| Full abstraction  | Partial abstraction |
| implements        | extends             |
| No object         | No object           |
| Multiple possible | Single inheritance  |

## What is Constructor?

Constructor initializes object.

## What is this keyword?

Refers to current object.

## What is super keyword?

Refers to parent class.

## What is Exception Handling?

Mechanism to handle runtime errors.

## What is Polymorphism?

One interface, many forms.

## What is Encapsulation?

Wrapping data and methods together.

## What is Abstraction?

Hiding implementation details.
